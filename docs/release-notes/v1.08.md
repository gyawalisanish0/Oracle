# Domain AI v1.08

**Local-first AI chat for Android. Your words stay on your device.**

Domain AI answers every message on-device by default, with zero network access. The
cloud is strictly opt-in — gated by a network kill switch and a redactor that strips
sensitive information from your prompt before it's sent to a cloud model.

## What's new since 1.05

- **Hugging Face model picker.** A new "Hugging Face models" section appears in
  Cloud Settings. Enter your HF API token, tap "Fetch models", and browse the top
  inference-ready text-generation models sorted by downloads. Tap "Use" to validate
  and activate — no custom URLs or manual model IDs needed.
- **Self-hosted Space backend.** You can now deploy `backend/` as a Hugging Face
  Docker Space to run a llama.cpp model you control. Point Domain AI's custom endpoint
  at your Space URL for a fully private, self-managed cloud backend. Supports a team
  mode (one Space, one shared access token, multiple phones) and is designed to be
  forked for community use.
- **Faster prompt processing on high-RAM devices.** The prompt-prefill batch size
  (N_BATCH) now scales to your device's RAM at startup rather than using a fixed
  default. Phones with ≥ 8 GB RAM process long prompts measurably faster;
  budget devices are unchanged.

## Highlights

- **Local by default** — replies generated on-device; no network unless you opt in
  per message.
- **Network kill switch** — outbound requests made impossible at one auditable
  chokepoint. On by default.
- **PII redaction** — emails, phone numbers, SSNs, cards and IPs are stripped before
  any cloud call; each cloud reply shows the exact redacted text that was sent.
- **Opt-in cloud** — connect OpenRouter, Hugging Face, or any OpenAI-compatible
  provider. Off by default; entirely your choice, per message.
- **Self-hosted backend** — run your own llama.cpp Space for maximum privacy and control.
- **Routing transparency** — every reply is badged On-device, Cloud, or Blocked.
- **Encrypted at rest** — conversations stored in an AES-256 file keyed by the Android
  Keystore; backups disabled; no analytics or trackers (crash reporting is opt-in, off
  by default).
- **GPU acceleration** — optional GPU offload with an in-app CPU-vs-GPU benchmark.
- **Adaptive performance** — inference threads, context length, and prompt-prefill
  batch size all scale to your device's hardware at startup. All three are user-
  configurable from Settings or the chat quick-panel.
- **Background generation** — long replies and downloads keep running when you leave
  the app, kept alive by a foreground service; swipe the app from recents to stop.
- **Reliable history** — conversations persist correctly across app restarts.

## Requirements

- Android 8.0 (API 26) or newer.
- An arm64-v8a device — the native runtime is arm64-only (universal among modern
  phones; older 32-bit-only devices are not supported).
- A GGUF model: download one in-app or import your own. Or configure a cloud provider
  (Hugging Face, OpenRouter, or a custom endpoint) to skip local model setup.

## Install

Download the APK below and sideload it (enable "Install unknown apps" for your
